# HEARTBEAT.md - 定期检查任务清单

**用途**: 心跳检查时自动执行的任务列表

---

## 📋 定期检查项

### 每 2 小时执行 (心跳检查时)

- [ ] 检查定时任务状态 (cron list)
- [ ] 验证核心任务是否正常运行
- [ ] 检查是否有失败任务需要重试
- [ ] 记录系统健康度

### 每日执行 (23:00)

- [ ] 创建/更新当日 memory 文件
- [ ] 更新技能记忆表格
- [ ] 执行自我学习升级任务
- [ ] 整理今日进化成果
- [ ] **运行缓存自动清理** (`tools/缓存自动清理.py`)
- [ ] **查看执行时间统计** (`tools/任务执行时间统计.py --analyze`)

### 每周执行 (周日 20:00)

- [ ] 生成每周进化报告
- [ ] 执行策略回测
- [ ] 清理过期临时文件

---

## 🎯 核心监控指标

| 指标 | 阈值 | 告警 |
|------|------|------|
| 任务执行率 | ≥95% | <80% 告警 |
| 自动修复率 | ≥80% | <50% 告警 |
| Token 余额 | 充足 | 不足时告警 |
| JWT 认证 | 正常 | 失败时告警 |

---

## 📞 汇报规则

- ✅ 正常状态：心跳检查后回复 HEARTBEAT_OK
- ⚠️ 轻微问题：记录到 memory 文件，不主动汇报
- 🔴 严重问题：立即向用户发送通知
