## ❌ 问题 1: 耗时显示错误

### 现象
```
**执行耗时**: 0.0 秒
```

### 原因
监控脚本只计算了"汇报生成"时间，没有计算：
- ❌ session 启动时间 (~500ms)
- ❌ 任务状态检查时间 (~10 秒)
- ❌ 自动修复时间 (~2 秒)
- ❌ 日志记录时间 (~1 秒)
- ❌ 通知发送时间 (~1 秒)

### 实际耗时
```
lastDuration: 15000ms (15 秒) ← cron 记录的真实耗时
```

### 修复方案
```python
# 修复前
start_time = time.time()
# ... 执行监控 ...
report = generate_report(start_time)  # 只计算汇报生成时间

# 修复后
total_start = time.time()
# ... 执行监控 ...
report = generate_report(total_start)  # 计算总耗时
print(f"**执行耗时**: {total_elapsed:.1f}秒")
```

## 🤔 问题 2: 这个任务的作用是什么？

### 当前功能

**每 2 小时自动执行**:
1. ✅ 检查 23 个定时任务状态

### 实际效果

**优点**:
- ✅ 自动发现问题

- ✅ 记录详细日志
- ✅ 定期汇报状态

**缺点**:
- ❌ 每 2 小时执行，频率较高
- ❌ 每次耗时 15 秒，消耗 Token
- ❌ 非交易时段也执行 (无效)
- ❌ 汇报内容重复 (每次都是"正常")

## 💡 优化建议

### 方案 1: 降低频率 (推荐)

**优化**: 每 2 小时 → 每 4 小时

| 指标 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 每日执行 | 13 次 | 6 次 | -54% |
| Token 消耗 | 基准 | -54% | 大幅节省 |
| 问题发现 | 2 小时内 | 4 小时内 | 可接受 |

**具体时间表**:
```
优化前：00:00, 02:00, 04:00, 06:00, 08:00, 10:00, 12:00, 14:00, 16:00, 18:00, 20:00, 22:00, 00:00
优化后：00:00, 04:00, 08:00, 12:00, 16:00, 20:00
```

---

### 方案 2: 时段限制 (强烈推荐)

**优化**: 仅在关键时段执行

| 时段 | 执行 | 说明 |
|------|------|------|
| 08:00 | ✅ | 盘前检查 (7:00 任务后) |
| 12:00 | ✅ | 午间检查 (上午任务后) |
| 16:00 | ✅ | 盘后检查 (15:30 任务后) |
| 23:30 | ✅ | 睡前检查 (23:00 任务后) |
| 其他 | ❌ | 不执行 |

**效果**:
- 每日执行：13 次 → **4 次** (-69%)
- Token 消耗：减少 69%
- 问题发现：关键时点检查

---

### 方案 3: 按需触发 (最佳)

**优化**: 只在以下情况执行

1. **任务失败时**: 某个任务连续失败≥3 次
2. **用户手动触发**: `openclaw cron run 心跳学习`
3. **每日固定**: 23:30 执行一次全面检查

**效果**:
- 每日执行：13 次 → **1-3 次** (-77%~-92%)
- Token 消耗：大幅减少
- 问题发现：更精准
