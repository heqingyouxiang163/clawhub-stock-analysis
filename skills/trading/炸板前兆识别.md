# 炸板前兆识别技能 v1.0

## 功能描述
识别涨停股炸板前的预警信号，提前撤离避免亏损。

## 炸板前 5 大信号

### 信号 1：封单骤降 ⚠️

| 指标 | 阈值 | 风险等级 |
|------|------|----------|
| 封单减少 30-50% | 1 分钟内 | 🟡 中风险 |
| 封单减少 50-70% | 1 分钟内 | 🟠 高风险 |
| 封单减少>70% | 1 分钟内 | 🔴 极高风险 |

**操作**：
- 🟡 准备卖出（挂单）
- 🟠 减仓 50%
- 🔴 立即清仓

---

### 信号 2：反复开板 ⚠️

| 开板次数 | 风险等级 | 操作 |
|----------|----------|------|
| 1 次 | 🟢 正常 | 观察 |
| 2 次 | 🟡 中风险 | 准备卖出 |
| 3 次+ | 🔴 高风险 | 立即清仓 |

**回封时间判断**：
- <5 分钟回封：✅ 真回封，持有
- 5-15 分钟回封：⚠️ 弱回封，减仓
- >15 分钟回封：❌ 假回封，清仓

---

### 信号 3：板块走弱 ⚠️

| 板块变化 | 风险等级 | 操作 |
|----------|----------|------|
| 板块涨停减少 1-2 家 | 🟢 正常 | 观察 |
| 板块涨停减少 3-5 家 | 🟡 中风险 | 减仓 |
| 板块涨停减少>5 家 | 🔴 高风险 | 清仓 |

**板块龙头炸板**：
- 龙一炸板 → 龙二龙三高危，立即卖出
- 板块指数跌>2% → 集体炸板风险，清仓

---

### 信号 4：大盘跳水 ⚠️

| 指数变化 | 风险等级 | 操作 |
|----------|----------|------|
| 指数跌 0.5-1% | 🟡 中风险 | 减仓 50% |
| 指数跌 1-2% | 🟠 高风险 | 减仓 80% |
| 指数跌>2% | 🔴 极高风险 | 全部清仓 |

**千股跌停预警**：
- 跌停家数>100 家 → 空仓
- 跌停家数>500 家 → 空仓 + 关软件

---

### 信号 5：成交量异常 ⚠️

| 成交量变化 | 风险等级 | 操作 |
|------------|----------|------|
| 成交量骤增 50% | 🟡 中风险 | 观察 |
| 成交量骤增 100% | 🟠 高风险 | 减仓 |
| 成交量骤增 200%+ | 🔴 极高风险 | 清仓 |

**量价背离**：
- 封单减少 + 成交量增加 = 主力出货
- 封单不变 + 成交量增加 = 分歧加大

## 炸板预警流程

### 盘中监控（每 5 分钟）

```
【检查清单】
□ 封单变化（对比 5 分钟前）
□ 开板次数（累计）
□ 板块涨停家数（变化）
□ 大盘指数（涨跌幅）
□ 成交量（对比均量）

【风险评分】
- 0-1 个信号：🟢 安全，持有
- 2 个信号：🟡 中风险，准备卖出
- 3 个信号：🟠 高风险，减仓 50%
- 4-5 个信号：🔴 极高风险，清仓
```

---

## 炸板应对策略

### 炸板前（预警阶段）

```
【🟡 中风险】
- 挂单卖出（涨停价）
- 准备撤单（如回封）
- 观察 5 分钟

【🟠 高风险】
- 减仓 50%
- 剩余挂单卖出
- 准备清仓

【🔴 极高风险】
- 立即清仓
- 不等待回封
- 保住利润/减少亏损
```

### 炸板后（应对阶段）

```
【真回封】<5 分钟
- 可持有
- 观察封单强度
- 再次炸板→清仓

【弱回封】5-15 分钟
- 减仓 50%
- 观察二次回封
- 不回封→清仓

【假回封】>15 分钟
- 立即清仓
- 不抱幻想
- 承认错误
```

## 炸板损失控制

### 止损规则

| 情况 | 止损位 | 操作 |
|------|--------|------|
| 打板当日炸板 | -3% | 次日开盘卖出 |
| 持仓股炸板 | -5% | 无条件止损 |
| 连续炸板 2 次 | -8% | 停止交易 3 天 |

### 心态控制

```
【炸板后】
✅ 承认错误，不抱幻想
✅ 立即止损，不等待
✅ 记录原因，避免重复
❌ 不追加资金补仓
❌ 不情绪化交易
❌ 不报复性交易
```

## 使用流程

### 盘中实时监控

```
【每 5 分钟检查】
1. 封单变化
2. 开板次数
3. 板块强弱
4. 大盘走势
5. 成交量变化

- 0-1 信号：持有
- 2 信号：准备卖出
- 3 信号：减仓 50%

```

### 炸板后处理

```
【当日炸板】
- 次日开盘卖出
- 不等待反包
- 承认错误

【记录复盘】
- 炸板原因
- 信号是否识别
- 如何避免重复
```
