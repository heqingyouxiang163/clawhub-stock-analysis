## 🎯 六、教训记录

### 核心教训

1. **预防优于修复**
   - Token 配额监控应该提前告警，而不是耗尽后修复
   - 建议：添加 Token 余额监控任务 (每日 8:00 检查)

2. **数据驱动决策**
   - 超时设置应基于历史执行时间统计
   - 建议：记录 7 天执行时间，计算 P95 作为基准

3. **平滑调度**
   - 避免任务集中执行导致资源峰值
   - 建议：任务时间分散，间隔至少 15 分钟

4. **容错设计**
   - 单点失败不应影响整体系统
   - 建议：关键任务增加重试 + 备用方案

5. **路径规范**
   - `~` 路径在不同上下文中解析不一致
   - 建议：强制使用绝对路径，启动时验证
