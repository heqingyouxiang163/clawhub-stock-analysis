# Gateway 重启检查协议

**创建时间**: 2026-03-17
**用途**: Gateway 重启后自动检查系统状态

---

## 触发条件

当检测到以下事件时触发检查:
- GatewayRestart 通知
- Session 异常中断
- 用户手动重启

---

## 检查清单

### 1️⃣ 检查 cron 任务状态

```bash
cron status
```

**预期结果**:
- enabled: true
- jobs: 22+ (包含所有任务)

**如异常**:
- 任务丢失 → 重建任务
- 状态 disabled → 启用任务

---

### 2️⃣ 检查关键文件

| 文件 | 检查项 | 预期 |
|------|--------|------|
| SOUL.md | 是否存在 | ✅ |
| MEMORY.md | 是否存在 | ✅ |
| AGENTS.md | 是否存在 | ✅ |
| memory/自我进化/ | 最新文件 | ✅ |

---

### 3️⃣ 检查心跳任务

**任务名**: 🦞 每小时心跳学习

**检查**:
- 是否存在
- 是否启用
- 下次运行时间

---

### 4️⃣ 检查角色设定

**读取 MEMORY.md**:
- 角色设定是否完整
- 策略库索引是否正确
- 知识库索引是否正确

---

### 5️⃣ 检查待办事项

**读取 memory/YYYY-MM-DD.md**:
- 是否有未完成的任务
- 是否有进行中的工作

---

## 恢复流程

```
检测 GatewayRestart
    ↓
执行检查清单
    ↓
发现问题
    ↓
自动修复
    ↓
写入恢复日志
    ↓
向用户汇报
```

## 自动修复脚本

### 重建丢失的 cron 任务

```bash
# 检查任务是否存在
cron list | grep "任务名"

# 如不存在，重建任务
cron add --job '{...}'
```

### 启用禁用的任务

```bash
# 启用任务
cron update --jobId <ID> --patch '{"enabled": true}'
```
