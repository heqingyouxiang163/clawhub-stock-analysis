## 🐛 问题描述

**原显示**:

- 实际耗时约 35 毫秒

## ✅ 优化方案

### 动态精度显示

根据耗时自动选择显示单位：

| 耗时范围 | 显示单位 | 示例 |
|----------|----------|------|
| < 1 秒 | 毫秒 | `2 毫秒` |
| 1-60 秒 | 秒 (2 位小数) | `0.04 秒` |
| > 60 秒 | 秒 (1 位小数) | `1.5 秒` |

### 代码实现

```python
# 根据耗时选择显示精度
if duration < 1:
    duration_str = f"{duration*1000:.0f}毫秒"
elif duration < 60:
    duration_str = f"{duration:.2f}秒"
else:
    duration_str = f"{duration:.1f}秒"
```

## 🔧 技术细节

### 时间戳 vs datetime

```python
# ✅ 推荐：使用 time.time()
start = time.time()
# ... 执行代码 ...
duration = time.time() - start

# ❌ 不推荐：使用 datetime.now()
start = datetime.now()
# ... 执行代码 ...
duration = (datetime.now() - start).total_seconds()
```

**原因**:
- `time.time()` 返回 Unix 时间戳，精度更高
- `datetime.now()` 可能受系统时钟调整影响
- `time.time()` 计算更简单快速

### 显示精度选择

```python
# 根据量级选择单位
if duration < 1:
    # 毫秒级 (<1 秒)
    duration_str = f"{duration*1000:.0f}毫秒"
elif duration < 60:
    # 秒级 (1-60 秒)
    duration_str = f"{duration:.2f}秒"
else:
    # 分钟级 (>60 秒)
    duration_str = f"{duration:.1f}秒"
```

## 💡 最佳实践

### 1. 选择合适的精度

- **快速操作**: 毫秒 (ms)
- **中等操作**: 秒 (s)，2 位小数
- **慢速操作**: 秒 (s)，1 位小数

### 2. 使用 time.time()

```python
import time

start = time.time()
# ... 执行代码 ...
duration = time.time() - start
```

### 3. 避免常见错误

```python
# ❌ 错误：使用 datetime 计算耗时
start = datetime.now()
duration = (datetime.now() - start).seconds  # 只返回整数秒

# ✅ 正确：使用 time.time()
start = time.time()
duration = time.time() - start  # 返回浮点数秒
```
