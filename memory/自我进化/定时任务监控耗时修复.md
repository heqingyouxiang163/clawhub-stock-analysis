## 🐛 问题描述

定时任务监控执行完成后，显示的耗时不正确，可能是：
- 显示的是真实时间差（包含等待时间）
- 而不是实际执行时间

---

## 🔍 根本原因

原代码使用 `datetime.now()` 计算耗时：

```python
# ❌ 错误代码
total_start = datetime.now()
end_time = datetime.now()
duration = (end_time - total_start).total_seconds()
```

**问题**: `datetime.now()` 计算的是真实时间差，如果程序中有 I/O 等待、网络请求等，会包含等待时间，导致耗时显示不准确。

---

## ✅ 修复方案

改用 `time.time()` 计算实际执行时间：

```python
# ✅ 正确代码
import time

total_start = time.time()  # 记录总开始时间 (使用 time.time() 计算真实耗时)
start_datetime = datetime.now()  # 用于显示的时间戳

# ... 执行任务 ...

end_time_unix = time.time()  # 记录总结束时间 (unix 时间戳)
end_datetime = datetime.now()  # 用于显示的时间戳
duration = end_time_unix - total_start  # 计算真实耗时 (秒)
```

**优势**:
- `time.time()` 返回 Unix 时间戳，精度高
- 计算的是实际经过的时间，不包含系统时钟调整
- 更适合性能监控

## 📝 相关知识

### time.time() vs datetime.now()

| 特性 | `time.time()` | `datetime.now()` |
|------|---------------|------------------|
| 返回值 | Unix 时间戳 (浮点数) | datetime 对象 |
| 用途 | 计算时间差、性能监控 | 显示时间、日期运算 |
| 精度 | 微秒级 | 微秒级 |
| 时区 | UTC | 本地时区 |
| 推荐场景 | **性能监控、耗时计算** | **时间显示、日志记录** |

### 最佳实践

```python
import time
from datetime import datetime

# 性能监控 → 使用 time.time()
start = time.time()
# ... 执行代码 ...
duration = time.time() - start
print(f"执行耗时：{duration:.2f}秒")

# 时间显示 → 使用 datetime.now()
timestamp = datetime.now().strftime('%Y-%m-%d %H:%M:%S')
print(f"执行时间：{timestamp}")
```
