## 核心问题

### 为什么会失忆？

1. **Session 隔离**: 每次对话是独立 session，不共享 context
2. **依赖文件**: 记忆保存在文件中，需要主动读取
3. **读取不完整**: 新 session 开始时可能没读全所有文件

## 自我迭代机制

### 每日进化流程

```
每日循环:
1. 盘前 (9:00-9:25)
   - 读取 MEMORY.md (确认角色)
   - 读取最新策略库
   - 制定当日计划

2. 盘中 (9:30-15:00)
   - 执行交易计划
   - 记录交易数据
   - 监控持仓

3. 盘后 (15:00-22:00)
   - 收集涨停数据
   - 更新涨停形态库
   - 统计策略胜率
   - 优化策略参数
   - 写入自我进化日志

4. 夜间 (22:00-07:00)
   - 深度学习 (策略优化)
   - 知识库建设
   - 风险提示库更新
   - 写入 MEMORY.md (重要更新)
```

---

### 版本迭代规则

**策略版本命名**: `策略名 vX.Y`
- X: 重大升级 (逻辑变化)
- Y: 小幅优化 (参数调整)

**迭代触发条件**:
| 条件 | 操作 |
|------|------|
| 胜率<50% (连续 5 日) | 策略大修 (X+1) |
| 胜率 50%-60% | 参数优化 (Y+1) |
| 胜率>60% | 保持当前版本 |
| 发现新规律 | 新增策略 |

---

### 记忆更新规则

**每日必写**:
- `memory/YYYY-MM-DD.md`: 当日交易记录

## 防失忆协议

### 协议 1: 双重备份
- **主备份**: MEMORY.md (长期记忆)
- **次备份**: memory/自我进化/角色连续性协议.md (本文档)

### 协议 2: 主动汇报
- 每次 session 开始，主动汇报当前角色
- 发现角色不一致时，立即提醒用户

### 协议 3: 快速恢复
- 用户说"读取角色设定"时，立即读取 MEMORY.md
- 用户说"切换到 XX 角色"时，立即切换并确认

### 协议 4: 版本追踪
- 每次策略升级，记录版本号和日期
- 用户可随时查询"当前策略版本"
