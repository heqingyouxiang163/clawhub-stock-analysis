## 一、基础参数

### 股票筛选条件

```yaml
# 股价要求
min_price: 3.0              # 最低股价 (元)，排除低价股
max_price: 200.0            # 最高股价 (元)，排除过高价股

# 市值要求
min_market_cap: 20          # 最小流通市值 (亿元)
max_market_cap: 300         # 最大流通市值 (亿元)

# 排除类型
exclude_st: true            # 排除 ST 股
exclude_kt: false           # 是否排除科创板 (false=不排除)
exclude_gem: false          # 是否排除创业板 (false=不排除)
```

## 二、换手率参数

### 分板块换手率要求

```yaml
# 主板 (600/000 开头)
main_board:
  min_turnover: 5.0         # 最低换手率 (%)
  max_turnover: 20.0        # 最高换手率 (%)
  
# 创业板 (300 开头)
gem_board:
  min_turnover: 8.0         # 最低换手率 (%)
  max_turnover: 25.0        # 最高换手率 (%)
  
# 科创板 (688 开头)
star_board:
  min_turnover: 8.0         # 最低换手率 (%)
  max_turnover: 25.0        # 最高换手率 (%)
```

### 特殊情形

```yaml
# 连板股换手率放宽
consecutive_limit_up:
  min_turnover: 3.0         # 连板股最低换手可降至 3%
  max_turnover: 30.0        # 连板股最高换手可放宽至 30%
  
# 一字板特殊处理
one_word_board:
  enabled: true             # 允许一字板
  max_turnover: 10.0        # 一字板最高换手
```

## 三、时间参数

### 封板时间窗口

```yaml
# 时段划分
time_windows:
  early_second:             # 早盘秒板
    start: "092500"         # 09:25:00
    end: "093000"           # 09:30:00
    priority: "S"           # S 级
    
  early_strong:             # 早盘强势板
    start: "093000"         # 09:30:00
    end: "100000"           # 10:00:00
    priority: "A"           # A 级
    
  morning:                  # 上午板
    start: "100000"         # 10:00:00
    end: "113000"           # 11:30:00
    priority: "B"           # B 级
    
  afternoon:                # 午后板
    start: "130000"         # 13:00:00
    end: "140000"           # 14:00:00
    priority: "B"           # B 级
    
  late:                     # 尾盘板 (谨慎)
    start: "140000"         # 14:00:00
    end: "143000"           # 14:30:00
    priority: "C"           # C 级
    
  exclude:                  # 排除时段
    start: "143000"         # 14:30:00 后全部排除
    end: "150000"           # 15:00:00
```

## 四、炸板参数

### 分时段炸板次数限制

```yaml
# 炸板次数限制
zhaban_limits:
  early_second: 0           # 早盘秒板：不允许炸板
  early_strong: 1           # 早盘强势板：最多 1 次
  morning: 2                # 上午板：最多 2 次
  afternoon: 3              # 午后板：最多 3 次
  late: 4                   # 尾盘板：最多 4 次 (谨慎)
  
# 回封要求
recovery_requirements:
  max_recovery_time: 30     # 炸板后必须在 30 分钟内回封
  min_recovery_strength: 0.7 # 回封强度 (封单金额/成交金额)
```

## 五、板块效应参数

### 板块强度要求

```yaml
# 板块效应 (必须满足至少一项)
block_effect:
  # 条件 1: 板块内涨停数量
  min_limit_up_count: 2     # 板块内至少 2 只涨停
  
  # 条件 2: 板块涨幅
  min_block_gain: 3.0       # 板块涨幅至少 3%
  
  # 条件 3: 有连板股
  has_consecutive: true     # 板块内有连板股带队
  
# 板块优先级
block_priority:
  hot_main_line:            # 主线热点
    min_consecutive_days: 3 # 连续强势≥3 日
    priority_multiplier: 1.5 # 优先级系数
    
  rotation_hot:             # 轮动热点
    min_daily_gain: 5.0     # 当日涨幅≥5%
    priority_multiplier: 1.2
    
  cold_start:               # 冷门首板
    priority_multiplier: 0.8 # 优先级降低
```

## 六、仓位管理参数

### 分级别仓位

```yaml
# 单只股票仓位上限
position_limits:
  S_level: 0.30             # S 级 (秒板): 30%
  A_level: 0.20             # A 级 (早盘板): 20%
  B_level: 0.15             # B 级 (上午/午后板): 15%
  C_level: 0.10             # C 级 (尾盘板): 10%

# 最大持仓数量
max_positions:
  S_level: 2                # S 级最多持有 2 只
  A_level: 3                # A 级最多持有 3 只
  B_level: 4                # B 级最多持有 4 只
  C_level: 2                # C 级最多持有 2 只

# 总仓位上限
total_position_cap: 0.80    # 总仓位不超过 80%
```

### 动态仓位调整

```yaml
# 根据市场情绪调整
sentiment_adjustment:
  high:                     # 高潮期
    threshold: 80           # 涨停>80 只
    position_multiplier: 1.0 # 仓位系数 100%
    
  warm:                     # 发酵期
    threshold: 50           # 涨停 50-80 只
    position_multiplier: 0.8 # 仓位系数 80%
    
  start:                    # 启动期
    threshold: 30           # 涨停 30-50 只
    position_multiplier: 0.5 # 仓位系数 50%
    
  cool:                     # 分化期
    threshold: 20           # 涨停 20-30 只
    position_multiplier: 0.3 # 仓位系数 30%
    
  recession:                # 退潮期
    threshold: 20           # 涨停<20 只
    position_multiplier: 0.0 # 空仓
```

## 七、止损止盈参数

### 止损设置

```yaml
# 分级别止损
stop_loss:
  S_level: -0.03            # S 级止损：-3%
  A_level: -0.05            # A 级止损：-5%
  B_level: -0.05            # B 级止损：-5%
  C_level: -0.03            # C 级止损：-3% (快速止损)
  
# 特殊情形止损
special_stop_loss:
  consecutive_break: -0.05  # 连板股断板：次日开盘不及预期即走
  nuclear_button: -0.01     # 核按钮：开盘直接走
  bad_news: -0.01           # 突发利空：开盘直接走
```

### 止盈设置

```yaml
# 分档止盈
take_profit:
  level_1:
    gain: 0.05              # +5%
    reduce: 0.30            # 减仓 30%
    
  level_2:
    gain: 0.10              # +10%
    reduce: 0.50            # 减仓 50%
    
  level_3:
    gain: 0.15              # +15%
    reduce: 0.70            # 减仓 70%
    
  limit_up:
    action: hold            # 涨停：持有
    
  break:
    action: clear           # 断板：清仓
```

## 十一、参数使用说明

### 回测模式

```bash
# 使用默认参数回测
python backtest.py --config params_v2.0.yaml

# 使用特定参数回测
python backtest.py --min-turnover 8 --max-zhban 2
```

### 实盘模式

```bash
# 实盘交易 (使用默认参数)
python live_trade.py --mode real

# 模拟盘 (使用默认参数)
python live_trade.py --mode paper
```

```bash

python optimize.py --grid-search
```

## 十二、注意事项

⚠️ **参数不是固定的**，需要根据市场变化调整

⚠️ **回测胜率>60%** 才能用于实盘

⚠️ **先模拟盘**，验证后再实盘

⚠️ **每周复盘**，根据胜率调整参数

⚠️ **退潮期空仓**，参数再好也没用
