## 📋 推送策略

### ✅ 新策略：每日统一推送

**定时任务**: 每天晚上 22:00 自动推送

**Cron**: `0 22 * * *`

**执行内容**:
```bash
cd /home/admin/openclaw/workspace/clawhub-stock-analysis
git add -A
git commit -m 'daily auto commit'
git push origin dev
```

**优点**:
1. ✅ 避免频繁推送 (减少 GitHub API 调用)
2. ✅ 累积一天的更改一起推送
3. ✅ 不用每次修改都手动推送
4. ✅ 自动化，无需人工干预

---

### ❌ 旧策略：改完就推

**问题**:
1. ❌ 推送太频繁 (一天几十次)
2. ❌ 浪费 GitHub API 配额
3. ❌ 需要手动执行
4. ❌ 提交记录太碎
