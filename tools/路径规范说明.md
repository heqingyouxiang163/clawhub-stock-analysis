# 📁 路径使用规范

**创建时间**: 2026-03-19 18:11  
**目的**: 避免写文件失败错误

---

## ❌ 错误写法

```python
# 会导致写文件失败
CACHE_FILE = "~/.openclaw/workspace/temp/cache.json"
path = "~/openclaw/workspace/memory/file.md"
```

**错误原因**: 
- Python 的 `open()` 函数不会自动展开 `~`
- 系统会尝试在字面意义的 `~` 目录下创建文件
- 导致 `FileNotFoundError` 或 `PermissionError`

---

## ✅ 正确写法

### 方法 1: 使用绝对路径 (推荐)

```python
CACHE_FILE = "/home/admin/openclaw/workspace/temp/cache.json"
path = "/home/admin/openclaw/workspace/memory/file.md"
```

**优点**: 
- 明确、无歧义
- 任何环境都能正常工作

---

### 方法 2: 使用 os.path.expanduser()

```python
import os

CACHE_FILE = os.path.expanduser("~/openclaw/workspace/temp/cache.json")
path = os.path.expanduser("~/openclaw/workspace/memory/file.md")
```

**优点**:
- 跨平台兼容
- 自动展开 `~` 为用户主目录

---

### 方法 3: 使用 pathlib (Python 3.4+)

```python
from pathlib import Path

CACHE_FILE = Path.home() / "openclaw" / "workspace" / "temp" / "cache.json"
path = Path.home() / "openclaw" / "workspace" / "memory" / "file.md"
```

**优点**:
- 现代化路径处理
- 跨平台兼容
- 链式调用更优雅

---

## 🚨 常见错误

### 错误 1: 直接在字符串中使用 `~`

```python
# ❌ 错误
with open("~/workspace/file.txt", "w") as f:
    f.write("content")

# ✅ 正确
with open("/home/admin/workspace/file.txt", "w") as f:
    f.write("content")
```

### 错误 2: 在配置文件中未展开 `~`

```json
// ❌ 错误
{
  "cache_path": "~/.openclaw/workspace/temp/cache.json"
}

// ✅ 正确
{
  "cache_path": "/home/admin/openclaw/workspace/temp/cache.json"
}
```
